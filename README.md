# Copilot BYOK Assistant

Languages: English | [简体中文](README.zh-CN.md) | [日本語](README.ja.md) | [한국어](README.ko.md)

A provider-neutral Codex skill for consulting GitHub Copilot CLI through a custom BYOK provider. This is a Codex skill; Copilot CLI is the external tool it invokes.

It does not include provider credentials, private endpoints, local shell aliases, or machine-specific paths. Configure your provider through environment variables or a local `.env` file.

## Use Cases

- **Codex-Copilot cross-development**: let Codex implement or refactor locally, then ask Copilot CLI through your BYOK provider to critique the plan, review the diff, or identify missing tests.
- **Dual-agent code review**: use one assistant as the primary implementer and the other as an independent reviewer before merging.
- **Provider comparison**: switch BYOK endpoints or model names to compare reasoning quality, code review strictness, latency, and cost without changing the Codex workflow.
- **Compliance and private routing**: route Copilot CLI through a provider, gateway, or network path approved by your team while keeping credentials outside the repository.
- **Debugging support**: pass failing command output to Copilot CLI for root-cause hypotheses, then verify the suggested checks locally.
- **Test and release planning**: ask for missing test cases, risky edge conditions, rollout checks, and regression areas after Codex prepares an implementation.

## Install

Clone this repository into your Codex skills directory:

```bash
mkdir -p ~/.codex/skills
git clone https://github.com/AlbertZhang520/copilot-byok-assistant.git ~/.codex/skills/copilot-byok-assistant
```

## Configure

```bash
cd ~/.codex/skills/copilot-byok-assistant
cp .env.example .env
$EDITOR .env
```

Required settings:

- `COPILOT_BYOK_BASE_URL`
- `COPILOT_BYOK_MODEL` or `COPILOT_BYOK_MODEL_ID`
- `COPILOT_BYOK_API_KEY` unless your provider does not require one

Check configuration:

```bash
./scripts/run-copilot-byok.sh --check
./scripts/run-copilot-byok.sh --print-config
```

## Use

```bash
./scripts/run-copilot-byok.sh -p "Review the current git diff for correctness bugs. Do not modify files." --silent
```

Or invoke the skill in Codex:

```text
Use $copilot-byok-assistant to consult my configured Copilot CLI provider on this implementation plan.
```

## Security

- Never commit `.env`, API keys, bearer tokens, private endpoints, or internal model names.
- If secrets were ever committed, create a fresh repository or clean history before publishing.
- Treat model output as advisory and verify with local evidence before editing code or reporting conclusions.
- `agents/openai.yaml` is Codex skill UI metadata generated by the skill template; it does not make the skill OpenAI-provider-specific.
